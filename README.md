# bilibili-uid-tool
Bilibili 用户信息采集工具  本项目是一个用于批量获取 Bilibili 网页端用户公开信息（如用户名、uid，账号有效状态）的 Python 自动化脚本。  该程序的核心设计目标是“稳定”与“数据安全”。为了应对目标网站严格的访问频率限制，脚本放弃了高并发访问，转而采用单线程低速模式，专为长时间、无值守，配置简单的后台挂机场景开发。程序内置了完整的异常处理流程，能够自动应对网络波动、账号临时冻结、本地文件误操作等突发状况。无论面临何种中断，程序都能确保已采集的数据安全保存至本地，且支持随时中断与无缝续传。

## 技术特性

1. **阻塞型重试机制**：请求遭遇拦截时，程序会暂停进入下一目标，当前进程原地挂起并执行退避休眠。只有当成功获取目标的确切信息后，循环才会继续，实现“零漏抓”。
      

2. **进程退出保护**：脚本捕获了 `KeyboardInterrupt` 异常。即使你手动中止进程（如 Ctrl\+C），程序也会强制将内存缓冲区内的数据安全写入本地文件后再关闭，确保数据零丢失。
      

3. **文件防毁坏读取**：在读取历史进度时，程序会自动尝试 `utf\-8\-sig`、`gbk` 和 `utf\-8` 三种编码。即使你用 Excel 修改并保存过数据文件，程序依然能正常识别断点。
      

4. **账号状态心跳核身**：设立 30 秒超时阈值。当长时间无响应时，看门狗会通过全局接口比对 Cookie 中的 UID 与服务器实际识别的 UID。若发现登录失效或网络断开，将立即紧急存盘并安全退出。

5. **动态退避算法**：统一捕捉 HTTP 412/403 及业务层 \-352/\-412 拦截信号。触发拦截后，执行 120 秒基数的阶梯递增休眠，并加入随机小数抖动，最大程度规避防火墙的机器特征检测。

## 常见问题与排错 \(Troubleshooting\)

### 1\. 终端进程长期卡死，没有新数据刷新

**原因**：Windows 终端的“快速编辑模式”在鼠标点击窗口内部时会冻结进程。

**解决**：点击终端窗口，按一下键盘的 Enter \(回车键\) 或 Esc 键，数据流将立即恢复。

### 2\. 连续提示目标被拦截并进入长时间休眠

**原因**：当前网络的公网 IP 触碰了防火墙阈值，被列入临时黑名单。

**解决**：
      按 Ctrl \+ C 退出程序（数据会自动保存）。切换网络（如手机开关一次飞行模式换 IP，或重启路由器）。重新运行程序即可。

### 3\. 程序提示“鉴权凭证已被注销”并自动退出

**原因**：网络中断或 Cookie 已过期。

**解决**：检查网络。若网络正常，需按照配置说明重新在浏览器获取最新的 Cookie 并更新代码。
